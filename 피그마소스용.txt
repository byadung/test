.github/workflows 는 깃허브 액션으로 자동 빌드 세팅한 결과 생성된 폴더니까 건드리지마
GitHub Actions로 자동 빌드 세팅하기
1단계: 빌드 설정 파일 추가하기
현재 GitHub 저장소 화면에서 Add file -> Create new file을 클릭합니다.

파일 이름 입력 칸에 아래 경로를 그대로 복사해서 붙여넣습니다. (슬래시를 입력하면 폴더가 자동으로 생성됩니다.)

Plaintext
.github/workflows/deploy.yml
파일 내용 입력 칸에 아래 코드를 그대로 복사해서 붙여넣습니다.

YAML
name: Deploy to GitHub Pages

on:
  push:
    branches: ["main"]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps: deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install pnpm
        uses: pnpm/action-setup@v3
        with:
          version: 9

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-code-version: 20
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install

      - name: Build
        run: pnpm run build

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: './dist'

      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
우측 상단의 Commit changes... 버튼을 누르고 한 번 더 Commit changes를 눌러 저장합니다.

2단계: Vite Base 경로 수정하기 (중요!)
GitHub Pages는 주소 뒤에 /test/처럼 저장소 이름이 붙기 때문에, 경로를 맞춰주어야 파일들을 제대로 불러옵니다.

저장소 메인 화면으로 돌아와 vite.config.ts 파일을 클릭합니다.

우측의 연필 아이콘(Edit this file)을 누릅니다.

기존 코드에 base: '/test/', 설정을 추가해 줍니다. 대략 아래와 같은 형태가 되어야 합니다.
import { defineConfig } from 'vite'
import path from 'path'
import tailwindcss from '@tailwindcss/vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  base: '/test/', // 깃허브 배포 경로를 원본 소스에 추가합니다.
  plugins: [
    react(),
    tailwindcss(),
    // Stub out Figma Make virtual modules so standalone builds succeed
    {
      name: 'figma-stubs',
      resolveId(id) {
        if (id === 'figma:foundry-client-api') return id
        if (id.startsWith('figma:asset/')) return id
      },
      load(id) {
        if (id === 'figma:foundry-client-api') return 'export default {}'
        if (id.startsWith('figma:asset/')) return `export default ''`
      },
    },
  ],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
  assetsInclude: ['**/*.svg', '**/*.csv'],
  build: {
    outDir: 'dist',
    emptyOutDir: true,
  },
})

3단계: Pages 배포 방식 변경하기
저장소 상단의 Settings 메뉴로 이동합니다.

왼쪽 메뉴에서 Pages를 클릭합니다.

Build and deployment 섹션의 Source 드롭다운을 클릭하여 기존 Deploy from a branch 대신 GitHub Actions로 변경합니다. (별도의 저장 버튼 없이 바로 변경됩니다.)
